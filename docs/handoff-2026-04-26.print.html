1. Project Summary

This repo is a single-page Vite + React trading desk for Hello Kitty Dream Village item trading.

It is not the same app shape as the TypeScript route-based Kimi sandbox app. During this handoff cycle, only the parts that cleanly mapped into the real repo were ported. The current app remains a single-page workspace with hash-based sections, a live/mock Supabase switch, catalog save flows, trade matching, and Dream Board scaffolding.

2. Tech Stack

  • React 19
  • Vite 8
  • @supabase/supabase-js
  • Plain CSS modules-by-file, imported from src/main.jsx

Main commands:

  • npm run dev
  • npm run build
  • npm run preview
  • npm run refresh:wiki-data

3. Environment

Supabase client config is read from env, not hardcoded:

  • VITE_SUPABASE_URL
  • VITE_SUPABASE_PUBLISHABLE_KEY
  • fallback: VITE_SUPABASE_ANON_KEY

Client setup lives in:

  • src/lib/supabaseClient.js

Do not replace the client file with literal keys. Set env vars instead.

Important note

The extra catalog seed migration for Kuromi + Cinnamoroll was added to the repo but was not executed locally in this session.

Dream Board schema note

src/hooks/useDreamBoard.js expects:

  • dream_boards
  • dream_board_stickers

The local supabase/schema.sql currently does not show those tables. That means the Dream Board UI is wired, but live save/load depends on those tables existing in the actual Supabase project.

If the live database does not have them yet, Dream Board persistence will fail until the schema is added.

9. Known Gaps / Risks

  1. Dream Board persistence may fail if dream_boards and dream_board_stickers are not present in Supabase.
  2. The extra catalog seed migration exists in git but has not been applied to the live database here.
  3. The current repo is intentionally not a full merge of the Kimi TypeScript app. It keeps the existing workspace shell and only ports compatible features.
